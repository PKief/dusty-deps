Features

Built for real-world dependency hygiene

A single command surfaces unmaintained packages lurking in your production dependencies.

⏱

Age-based detection

Flags dependencies whose latest npm release exceeds a configurable threshold. Default: 3 years.

☑

Allowlist with reasons

Intentionally old packages get documented in a config file — not silently ignored. Every exception has a reason.

⚙

Zero runtime dependencies

Uses only Node.js built-ins. The package that checks your dependency hygiene doesn't add to the problem.

→

CI-ready exit codes

Exit 0 when all pass, exit 1 when any fail. Drop it into any pipeline as a quality gate.

{}

JSON output

Machine-readable output with --json for programmatic consumption and custom reporting.

⧉

Importable API

Use check() directly in your own tools. Full TypeScript types included.

Usage

Three steps to clean dependencies

  1. Install

    Add it as a dev dependency or install it globally.

  2. Run

    Execute dusty-deps in any project with a package.json. It checks every production dependency against the npm registry.

  3. Decide

    Update stale packages, or add them to the allowlist with a documented reason. No silent tech debt.

CLI options
# default — 3 year threshold $ dusty-deps   # custom threshold (2 years) $ dusty-deps --threshold 730   # JSON for CI pipelines $ dusty-deps --json   # check a different project $ dusty-deps --cwd ../other-project
Programmatic API
import { check } from "dusty-deps";   const result = await check({ cwd: "./my-project", threshold: 1095, });   console.log(result.failed); // 3
Configuration

Document every exception

The allowlist isn't just a skip list — it's a record of deliberate decisions. Every old dependency gets a reason.

Config files (priority order)

  • dusty-deps.config.json
  • dusty-deps.config.mjs
  • .dusty-depsrc.json
  • "dusty-deps" in package.json
// dusty-deps.config.json { "threshold": 1095, "allowlist": { "lodash": "Stable utility library, infrequent releases", "path": "Node.js built-in polyfill, no updates expected", "jszip": "Mature ZIP library, no known vulnerabilities" } }
